Every assistant combines these components. **Intents and triggers** (how workflows start) are documented separately in [Triggers & Intents](/triggers-intents/overview) because they vary by solution type.

## Components

| Component | Role |
| --- | --- |
| **Agent Graphs** | Logic and orchestration layer — designed in Graph Studio |
| **Tools** | Action layer for API calls, integrations, and custom code |
| **Builds** | Versioned snapshots for deployment across environments |
| **Environments** | Dev, UAT, and Prod isolation with separate variables and logs |
| **Channels** | Where assistants interact — chat, voice, email |
| **Agent Registry** | Expose graphs as A2A agents and compose them from the canvas |

## Entry points (by solution type)
